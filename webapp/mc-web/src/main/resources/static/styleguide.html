Palette test — world-keeper's logbook

Real components on the live design tokens — a reference for judging future UI changes. Roles: lapis = act/links, grass = done, gold = warn, redstone = danger, dirt = structure.

Tokens & roles

Each colour has one job. The blue↔yellow anchors (lapis, gold) stay distinct under red-green colour vision; green/red states are reinforced with icons, never colour alone.

Buttons

Primary = solid lapis fill. Secondary = ink-on-paper (no colour). Lapis is deliberately distinct in hue from the redstone danger button.

Badges

Status pills. In-progress borrows the quiet lapis; done is grass; blocked is redstone.

Callouts

Info uses the quiet lapis (pale rule + ⓘ icon). Success / warning / danger each carry an icon as well as colour, so the state survives red-green colour vision.

Info. Resource graph last rebuilt 3 minutes ago.

✓

⚠

Warning. Two recipes have unresolved ingredients.

✗

Danger. Could not reach the data server.

Progress

Completion bars use the grass/status fill; a fully complete bar uses the brighter complete green.
